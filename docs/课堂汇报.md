# 课堂汇报：基于 21 关键点 MLP 的实时手势控制系统

> 计算机视觉课程项目 · 汇报材料

## 1. 项目动机

### 1.1 痛点
传统人机交互依赖鼠标键盘，在以下场景里很别扭：
- 演示 PPT 时手离讲台远
- 厨房做菜时手脏
- 残障用户无法使用键鼠
- VR/AR 场景中物理键鼠不可用

### 1.2 已有方案的不足
| 方案 | 缺点 |
|---|---|
| 蓝牙翻页器 | 功能单一，只能切下一页 |
| Leap Motion / Kinect | 需要专用硬件 |
| 语音控制 | 公共场合不便 |
| 现成手势 SDK（MediaPipe Gestures） | 类别固定（7 种），不能自定义动作绑定 |

### 1.3 本项目目标
1. **零额外硬件**：只用一颗普通摄像头
2. **真实控制**：能动鼠标、点击、滚动、调音量、切歌、切窗口
3. **可定制**：用户能录自己的手势数据，训自己的模型
4. **跨会话**：Wayland / X11 通吃
5. **轻量**：模型 < 100 KB，浏览器内推理 < 5 ms

### 2.1 为什么这样切分

| 职责 | 在哪里 | 理由 |
|---|---|---|
| 摄像头采集 | 浏览器 | 用户不愿把视频送到后端 |
| 手部关键点 | 浏览器 | MediaPipe Tasks Vision 在浏览器里 < 10 ms |
| MLP 推理 | 浏览器 | ONNX Runtime Web < 2 ms，零网络延迟 |
| 时间平滑 / 重排 | 浏览器 | 数据已在浏览器，省去往返 |
| 真实键鼠注入 | Python 后端 | 浏览器无法操作系统级输入 |

## 3. 算法设计

### 3.1 特征工程

输入：21 个手部关键点（每个点 (x, y, z) ∈ [0, 1]）

```python
def landmarks_to_feature(landmarks, handedness):
    # Step 1: 平移到 wrist 原点
    centered = landmarks - landmarks[0]
    
    # Step 2: 按手掌尺度缩放（去除距离影响）
    scale = max(‖wrist - middleMcp‖, ‖indexMcp - pinkyMcp‖)
    scaled = centered / scale
    
    # Step 3: 镜像归一（统一为右手坐标系）
    if handedness == "Left":
        scaled[:, 0] *= -1
    
    return scaled.flatten()  # (63,)
```

#### 不变性保证（已通过单元测试验证）
- ✅ 平移不变：手在画面任何位置识别都一致
- ✅ 尺度不变：手离镜头远近不影响
- ✅ 镜像归一：左右手用同一模型识别
- ✅ Python / JS 实现差异 < 6×10⁻⁸（保证训练 == 推理）

### 3.2 模型架构

```python
GestureMLP(
    Linear(63, 128) + ReLU + Dropout(0.2)
    Linear(128, 64) + ReLU + Dropout(0.2)
    Linear(64, 10)                            # 10 类静态手势
)
# 总参数：17,098（< 50 KB ONNX）
```

为什么 MLP 而不是 CNN/RNN：
- 关键点已经是高度抽象的语义特征，CNN 的空间卷积无用武之地
- 单帧静态识别，不需要 RNN 的时序建模
- 极小的模型 = 快速浏览器加载 + 微秒级推理

### 3.3 时间平滑

单帧识别即使准确率 99%，每秒 30 帧也意味着每秒约 0.3 帧抖动。所以加：

```javascript
// 滑窗投票：最近 7 帧出现频次最高的标签
// 滞后阈值：进入需要 ≥ 55% 占比，退出 < 40% 才放弃
class LabelSmoother {
    push(label, confidence) {
        this.history.push({label, confidence});
        if (this.history.length > 7) this.history.shift();
        const counts = countBy(this.history, "label");
        const [bestLabel, bestCount] = topEntry(counts);
        const ratio = bestCount / this.history.length;
        if (this.currentLabel !== bestLabel && ratio >= 0.55) {
            this.currentLabel = bestLabel;
        } else if (this.currentLabel === bestLabel && ratio < 0.4) {
            this.currentLabel = null;
        }
        return this.currentLabel;
    }
}
```

效果：用户切换手势时延迟约 100-200 ms，但识别非常稳定。

### 3.4 动态手势

静态 MLP 看不见运动信息。我们用平行通路检测「手腕轨迹」：

```javascript
class SwipeDetector {
    observe({x, y}, ts) {
        this.points.push({x, y, ts});
        // 保留最近 0.4 秒的点
        this.points = this.points.filter(p => ts - p.ts < 400);
        if (this.points.length < 5) return null;
        const dx = this.points.last.x - this.points.first.x;
        const dy = this.points.last.y - this.points.first.y;
        if (Math.hypot(dx, dy) < 0.22) return null;
        if (Math.abs(dx) > Math.abs(dy)) {
            return dx > 0 ? "swipe_right" : "swipe_left";
        }
        return dy < 0 ? "swipe_up" : "swipe_down";
    }
}
```

扩展了 4 个动态手势类，专门绑定到滚动 / 切歌等连续操作。

### 3.5 后处理 reranker

实测发现两个 MLP 易混类对：
1. **fist ↔ thumbs_up / thumbs_down**：四指都收拢，外形相似
2. **pinch ↔ ok / fist**：拇指食指距离都接近

针对这两组，加几何规则后处理。例如 fist→thumbs_up 校验：

```javascript
function rerankFistVsThumbs(result, landmarks) {
    if (result.label !== "fist") return result;
    const thumbTip = landmarks[4];
    const otherTips = [8, 12, 16, 20].map(i => landmarks[i]);
    const upGap = (min(otherTips.map(p => p.y)) - thumbTip.y) / palmScale;
    if (upGap > 0.35 && thumbExtension > 1.05) {
        return {...result, label: "thumbs_up"};
    }
    return result;
}
```

效果：在不重训模型的前提下，把 thumbs_up 的实测准确率从 ~70% 提到 ~95%。

## 4. 数据集与训练

### 4.1 数据采集

向导式 Web 页面 (`record.html`)：
- 10 个手势 × 250 帧 = 2500 个样本
- 每帧只存 21×3 = 63 个浮点 → 整套数据 < 1 MB
- 用户 5 分钟录完，断点续传

录制规范要求用户：
- 每个手势 8 秒里**轻微变化**手部位置/角度/远近
- 增强模型对真实场景的泛化能力

### 4.2 训练配置

| 超参数 | 值 |
|---|---|
| Optimizer | AdamW |
| 学习率 | 1×10⁻³ |
| Weight decay | 1×10⁻⁴ |
| Scheduler | CosineAnnealing |
| Batch size | 64 |
| Epochs | 80 |
| 数据增强 | 高斯抖动 σ=0.01 |
| 验证比例 | 15% |

### 4.3 实验结果

在我自录的 250 帧 × 10 类数据上：

| 指标 | 数值 |
|---|---|
| 训练时间 | 56.9 秒（CPU） |
| 验证准确率 | 100.0% |
| Macro-F1 | 1.000 |
| 推理延迟（浏览器 ONNX） | < 2 ms |
| 端到端延迟（摄像头→指令） | < 30 ms |

> 验证集 100% 听起来过拟合，但因为我们的特征已经做了几何归一化，
> 同一手势的特征向量本身就高度集中。要测真实泛化效果，应该收集
> 「不同时间、不同灯光、不同人」的 hold-out 集——这是后续工作。

### 4.4 多模型对比

`--name` 参数让训练 / 导出按子目录隔离，方便 A/B 测试不同方案：

```
outputs/gesture_mlp/
├── default/best.pth        ← 你录的标准数据
├── lowlight/best.pth       ← 暗光环境数据
├── synthetic/best.pth      ← 仅合成数据
└── augmented/best.pth      ← 加大增强强度
```

## 5. 安全设计

### 5.1 多重保险

| 机制 | 触发 | 行为 |
|---|---|---|
| 默认锁定 | 启动时 | 必须 UI 点「解锁」 |
| 测试模式 | UI 勾选 | 识别照常，不发动作 |
| 演示模式 | 后端不可达 | 自动进入测试模式且无法关闭 |
| FAILSAFE | 鼠标拖到屏角 | 抛异常 → 后端自动暂停 |
| Esc 物理键 | 任何时刻 | 用户可绑定为锁定动作 |
| 动作 cooldown | 每动作独立 | 防误触发洪水 |
| WS 断开 | 浏览器关 | 后端不再接收事件 |

### 5.2 隐私保护

- 视频流**不离开浏览器**
- WebSocket 只传 21 个关键点（63 个浮点）
- 无云端依赖（除了 MediaPipe CDN 模型，可缓存）

## 8. 创新点小结

1. **首次把「轻量 MLP + 关键点」推到浏览器里实时跑**：模型 < 100 KB，推理 < 2 ms
2. **跨语言特征对齐**：双语言实现一份逻辑，用 fixture 测试保证训练 ≡ 推理
3. **Reranker 后处理**：在不重训模型情况下修复 MLP 的系统性混淆
4. **混合静态 + 动态**：单帧 MLP + 轨迹分析串联，覆盖 14 类手势
5. **evdev 跨显示协议**：注入发生在显示服务器之下，Wayland / X11 通吃
6. **演示模式优雅降级**：云端访问无后端时仍能识别 + 展示 UI

## 9. 局限与未来工作

### 当前局限
- 仅支持 Linux（Wayland / X11）；Windows 后端待开发
- 单手识别（MediaPipe 配置 num_hands=1）
- 滑动手势靠简单线性轨迹，复杂动态手势（画圈/抖动）不支持
- 验证集来自录制时段，**不是真实分布**——长期使用准确率有待统计

### 未来工作
- [ ] **跨会话泛化测试**：在不同时段、灯光下采集 hold-out 集
- [ ] **小样本在线学习**：用户在 UI 里直接录 5 帧就能加新手势
- [ ] **双手协同**：捏 + 张 = 缩放 / 旋转 3D 物体
