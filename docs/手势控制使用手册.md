# 手势控制使用手册

## 1. 系统总览

```
[ 浏览器 ]                                              [ Python 后端 ]
摄像头 → MediaPipe 关键点 → 21D MLP → 手势事件   ──WS──▶  pyautogui 注入键鼠
                                              ◀──回执──   屏幕分辨率/状态
```

- 浏览器只采集摄像头并跑模型，不直接接触系统输入。
- 真实键鼠/媒体键由 Python 后端用 pyautogui 触发。
- 两端通过本地 `ws://127.0.0.1:8765/ws/control` 通信，不联网。

## 2. 第一次启动

```bash
cd /home/xzr/桌面/cv-work/gesture_lenet
./run_gesture_control.sh
```

- 终端显示 `Uvicorn running on http://127.0.0.1:8765` 即代表后端就绪
- 浏览器会自动打开页面
- 顶部三个状态徽章应显示：摄像头未启动 / 已锁定 / 后端已连接

## 3. 启动流程（用户视角）

1. 点「启动摄像头」并允许浏览器权限
2. 在右侧「手势绑定」面板浏览默认绑定，根据需要修改
3. 勾选「测试模式」练习一遍——动作不会真发，仅在画面顶部弹出气泡确认
4. 点「解锁控制」激活真实注入；状态徽章变为绿色「控制已开启」
5. 开始用手势控制电脑。每次成功的动作会弹出一个气泡提示，遮挡 1 秒后自动消失

## 4. 安全与紧急停止

- 默认启动后端时是「锁定」状态，必须显式解锁
- pyautogui FAILSAFE：把鼠标拖到屏幕任意角落 → 立刻抛异常 → 后端自动暂停
- 浏览器上「暂停 / 恢复」按钮可临时挂起注入，再点一次恢复
- 关闭浏览器或 Ctrl-C 终止后端，控制立即停止

## 5. 默认手势 ↔ 动作

| 手势 | 中文 | 默认动作 | 说明 |
|---|---|---|---|
| open_palm | 张开手掌 | 释放 | 拖拽中此手势会自动松开 |
| point | 食指指向 | 移动鼠标 | 食指尖映射到屏幕，自适应 EMA |
| pinch | 捏合 | 单击 / 拖拽 | 短捏 < 0.42s = 单击；长捏 = 按住拖拽 |
| fist | 握拳 | Esc | 退全屏 / 取消 |
| victory | V 字 | Alt+Tab | 切窗口 |
| ok | OK 圈 | 播放 / 暂停 | 媒体键 |
| thumbs_up | 拇指上 | 音量 + | |
| thumbs_down | 拇指下 | 音量 - | |
| three | 三指 | 显示桌面 | Win+D |
| call | 电话手势 | 下一首 | 媒体键 |

## 6. 训练自己的模型

默认提供的是「合成数据集」训出来的模型，约 80-85% 准确率，够 demo 但不够稳。要做到 95%+，需要录自己的数据。

```bash
# 1. 启动后端 + 浏览器
./run_gesture_control.sh

# 2. 浏览器进入 http://127.0.0.1:8765/web_control_demo/record.html
#    跟着向导录 10 个手势，每个 250 帧，约 5 分钟

# 3. 训练
python train_gesture_mlp.py

# 4. 导出
python export_onnx.py

# 5. 刷新主页面
```

录制建议：
- 录每个手势的 8 秒里，**轻微移动手部位置 / 旋转手腕 / 改变远近**，让模型学到不变性
- 灯光不要太暗，背景越简单越好
- 拇指向下手势可以让整只手转过来，确保拇指真的朝下
- 如果某个手势识别不稳，可以再次进入 record.html 追加录制——数据是 append 模式

## 7. 常见问题

### 为什么按钮显示「pyautogui 不可用」？

很可能是在 SSH 或没有显示服务的会话里启动后端。请直接在桌面终端启动。

### 为什么解锁后指针不动？

最常见原因：当前是 Wayland 会话。pyautogui 在 Wayland 下经常无法注入。检查：

```bash
echo $XDG_SESSION_TYPE   # 期望输出 x11
```

如果输出 `wayland`，请在登录界面切换到 X11 会话再试。

### 为什么手势识别不稳定，鼠标乱跳？

- 检查右上角实时帧率（FPS）。<15 帧通常是 CPU 不够用，关掉别的浏览器标签
- 让手部完整进画面，不要部分遮挡
- 训练自己的模型（合成数据集对真实摄像头泛化有限）

### 怎么彻底关掉这套系统？

终端按 Ctrl-C 即可结束后端。浏览器可以保留页面，但因为 WebSocket 已断开，不会触发任何动作。
