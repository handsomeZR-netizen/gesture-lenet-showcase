# 控制模式与模型说明

## 当前到底在用什么

当前项目已经拆成三种模式，但默认入口已经改成浏览器中文展示端。

## 1. 默认模式：中文 Web 展示端

- 入口：`./run_camera_demo.sh`
- 页面：`web_control_demo/index.html`
- 核心识别方式：浏览器摄像头 + `MediaPipe Web`
- 控制策略：手部关键点 + 规则状态机
- 展示界面：全中文答辩科技看板

这一模式：

- 不使用 `outputs/final_train/best_model.pth`
- 不做字母分类
- 不尝试把你的手势映射成 `A/B/C/...`
- 直接把手部动作映射成浏览器里的 3D 交互语义
- 没有 OpenCV 原生窗口标题乱码问题，更适合展示

当前支持的语义：

- 单手张开：准备状态
- 单手捏合：拖动球体
- 双手同时捏合：缩放并旋转球体
- 双手同时张开并保持：重置球体

如果浏览器没有摄像头权限，页面会直接给出中文提示；如果没有检测到手，状态栏和覆盖提示会明确提示当前原因。

## 2. 备用模式：Python 手势控制计算机

- 入口：`./run_showcase_demo.sh`
- 脚本：`infer_camera.py --mode showcase --control-mode mouse`
- 核心识别方式：`MediaPipe Hand Landmarker`
- 控制策略：手部关键点 + 规则状态机
- 展示界面：OpenCV 本地教学看板

这一模式仍然保留真实鼠标/键盘控制能力：

- `Open Palm`：准备状态，不触发桌面动作
- `Point`：移动鼠标指针
- `Pinch`：短按点击，长按拖拽
- `Victory`：切换窗口
- `Fist`：重置状态

如果缺少 `pyautogui`，或当前桌面环境阻止输入注入，程序会降级成只展示识别和控制状态的演示模式。Wayland 环境下如果指针不动，建议答辩前切换到 X11 会话。

## 3. 旧版模式：字母分类演示

- 入口：`./run_legacy_letter_demo.sh`
- Python 脚本：`infer_camera.py`
- 模型：`outputs/final_train/best_model.pth`

这一模式来自公开数据集 `sign-language-mnist`，本质上是课程型静态分类演示：

- 更适合灰度、居中、背景简单的手部图像
- 不适合直接拿来做自然场景交互
- 会和 PDF 里的某些示意图存在标签体系差异

## 为什么现在不再把旧模型当主线

原因很简单：

- 你真正想做的是“手势交互”，不是“字母考试”
- 球体拖动、缩放、旋转是连续控制，不适合用单帧字母分类来实现
- 旧模型即使继续修，也还是会受数据集分布限制

所以当前默认主线固定为：

```text
浏览器摄像头 -> MediaPipe 手关键点 -> 手势规则 -> 中文展示看板 + 3D 交互
```

## 你自己的模型现在到哪一步

当前已经完成的是“把旧模型从主线移开”。

后续如果要训练你自己的控制模型，推荐顺序是：

1. 先确定离散命令标签
2. 采集你自己的手势数据
3. 训练控制命令分类模型
4. 把它作为规则系统的补充，而不是替代连续关键点控制

更合适的自定义标签不是字母，而是：

- `grab`
- `release`
- `reset`
- `mode_switch`
- `color_toggle`

这类语义和交互目标是对齐的。

## 当前结论

- 现在默认运行时，不是旧模型
- 现在默认运行时，是关键点规则控制
- 旧模型还在，但已经是 `legacy`
- 你的“自己训练的控制模型”还没开始采数，这一步需要单独补
